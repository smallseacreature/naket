BACnet/IP BVLC messages begin with a one-byte type, one-byte function, and two-byte total length. 
Type 0x81 identifies BACnet/IP, and function 0x0A identifies an Original-Unicast-NPDU.

BACnet — Building Automation and Control Network
The overall protocol used by building systems such as HVAC, lighting, access control, and alarms.
B/IP — BACnet/IP
BACnet carried over IP networks, usually over UDP.
BVLL — BACnet Virtual Link Layer
The part of BACnet that adapts BACnet messages to a specific network technology such as IPv4.
BVLC — BACnet Virtual Link Control
The header and control messages used by the BVLL for BACnet/IPv4. It includes the type, function, and message length.
BBMD — BACnet Broadcast Management Device
A BACnet device that forwards BACnet broadcast traffic between IP subnets. Normal IP routers do not forward broadcasts, so BBMDs provide that functionality.
BDT — Broadcast Distribution Table
A list stored by a BBMD containing other BBMDs or subnet destinations that should receive forwarded broadcasts.
FDT — Foreign Device Table
A list stored by a BBMD containing remote BACnet devices that have registered to receive BACnet broadcasts.
FD — Foreign Device
A BACnet device located outside the local BACnet/IP subnet that registers with a BBMD.
NPDU — Network Protocol Data Unit
The BACnet network-layer portion of the message. It contains routing information, network addresses, priority, and related flags.
APDU — Application Protocol Data Unit
The application-layer BACnet message. It identifies operations such as ReadProperty, WriteProperty, Who-Is, or I-Am.
PDU — Protocol Data Unit
A general networking term for a block of data associated with a protocol layer.
TTL — Time To Live
In foreign-device registration, this tells the BBMD how long the registration remains valid, usually in seconds.
ACK — Acknowledgment
Indicates that a request was received or completed.
NAK — Negative Acknowledgment
Indicates that a request failed or was rejected.
BDT ACK — Broadcast Distribution Table Acknowledgment
A response containing the BBMD’s BDT entries.
FDT ACK — Foreign Device Table Acknowledgment
A response containing the BBMD’s FDT entries.


An NPDU begins with a minimum two-byte header:

Byte 0: NPDU version
Byte 1: NPDU control flags

The control byte contains several flags:

Bit 7:    Network-layer message
Bit 5:    Destination address information present
Bit 3:    Source address information present
Bit 2:    Reply expected
Bits 1–0: Message priority

A maximally populated BACnet NPDU would look like this:

NPDU
├── Version                         1 byte
├── Control                         1 byte
│
├── DNET                            2 bytes   if destination info is present
├── DLEN                            1 byte
├── DADR                            DLEN bytes
│
├── SNET                            2 bytes   if source info is present
├── SLEN                            1 byte
├── SADR                            SLEN bytes
│
├── Hop Count                       1 byte    if destination info is present
│
└── Payload
    ├── APDU                                  if Control bit 7 = 0
    └── Network-layer message                 if Control bit 7 = 1
